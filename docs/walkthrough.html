Overview

What is this?

Website Chat lets you point at any URL, crawl its pages with a real browser, index the content into a local vector store, then have a grounded AI conversation with that site's content — no cloud storage, no hallucination, citations included.

The golden rule: answers are grounded strictly in what was crawled. If the site doesn't say it, the AI won't say it either.

The system prompt has a hard rule: answer only from the CONTEXT chunks. If the context doesn't cover it, the AI says so — no hallucination.
🎨 Brand analysis bonus step

After indexing, the first 5 pages are sampled and sent to the LLM to extract a brand profile — formality level, key terms, writing patterns, voice summary. This is then injected into every chat system prompt so answers sound like the site, not like a generic chatbot.

Tech stack

Everything used

Chosen to keep everything local — no cloud vector DBs, no paid embeddings.

FastAPI
Backend framework

Async Python API with SSE streaming for both crawl progress and chat tokens.

Playwright
Web crawler

Real Chromium browser — renders JS-heavy SPAs, handles dynamic content.

hnswlib
Vector store

Local HNSW index — fast approximate nearest-neighbour search, zero cloud cost.

sentence-transformers
Embeddings

all-MiniLM-L6-v2 — 384-dim semantic vectors, runs entirely on CPU.

xAI / Grok
LLM

Powers chat responses and brand analysis via an OpenAI-compatible streaming API.

React + Vite
Frontend

TypeScript, Tailwind v4, component-first. Vite proxies /api to FastAPI.

Zustand
State management

Lightweight global store for sessions, active chat, and streaming message state.

SSE (EventSource)
Streaming protocol

Server-Sent Events stream crawl progress and chat tokens in real-time — no WebSockets needed.

pydantic-settings
Config

Typed environment variable loading from .env — API keys never touch the code.

Design decisions

Why these choices?

BFS crawl with Playwright crawler

Simple fetch() misses JS-rendered sites (React, Vue, Angular apps). Playwright runs a real browser so anything a human can see, the crawler can read. BFS keeps crawl order predictable and depth-bounded.

Recursive text chunking indexer

Splits on \n\n → \n → ". " → " " hierarchy with 500-char chunks and 100-char overlap. The overlap ensures a sentence cut across chunk boundaries can still be retrieved by either half.

hnswlib over a cloud DB indexer

Everything is local. No Pinecone account, no network latency, no data leaving the machine. Each crawl session gets its own HNSW index file on disk — delete a session, delete the file.

Fetch-based SSE for chat frontend

Chat uses fetch() with a ReadableStream instead of EventSource, because EventSource only supports GET requests. This lets us POST the full conversation history.

AbortController + 60s timeout frontend

Every chat request is wired to an AbortController. If the stream stalls for 60 seconds or the user navigates away, the request is cleanly cancelled and isStreaming always resets — no stuck spinners.

Tracking param stripping crawler

URLs with ?utm_source=twitter and ?utm_source=email are the same page. Stripping known tracking params (utm_*, fbclid, gclid, etc.) before deduplication prevents crawling the same page 10 times.

API reference

Endpoints

POST /api/crawl

Start a crawl. Body: { url, max_pages, max_depth }. Returns { job_id, session_id, status } immediately — crawl runs in background.

GET /api/crawl/{job_id}/status

SSE stream. Emits { type: "progress", status, pages_crawled, pages_total, current_url } until status is "complete" or "error".

POST /api/chat

SSE stream. Body: { session_id, message, conversation_history }. Emits token → sources → done events.

GET /api/sessions

Returns list of all indexed sessions: { session_id, site_url, site_name, pages_indexed, chunks_indexed, created_at }.

DELETE /api/sessions/{session_id}

Removes the session from sessions.json and deletes the hnswlib index from disk.

Getting started

Run it locally

Backend

cd backend → copy .env.example to .env → add your xAI key → pip install -r requirements.txt → playwright install chromium → python run.py

Frontend

cd frontend → npm install → npm run dev → open localhost:5173

Or just run ./start.sh from the repo root to start both at once.
